# Security Policy

## Supported Versions

| Version | Supported |
| ------- | --------- |
| 1.0.x   | Yes       |

## Reporting a Vulnerability

If you discover a security vulnerability in DFIR-Updater, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, please use one of the following methods:

1. **GitHub Security Advisory**: Use the [private vulnerability reporting](https://github.com/jameshenning/DFIR-Updater/security/advisories/new) feature on this repository.
2. **Email**: Contact the maintainer directly through their GitHub profile.

### What to Include

- A description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Suggested fix (if any)

### Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial assessment**: Within 1 week
- **Fix or mitigation**: Depends on severity, but high-severity issues will be prioritized

## Integrity Notice

DFIR-Updater is a digital forensics tool. Maintaining the integrity of forensic processes is critical. If you identify any behavior that could compromise forensic integrity (e.g., unintended file modifications, write operations to evidence drives), please report it immediately using the methods above.

## Verification

Always download releases from the [official GitHub Releases](https://github.com/jameshenning/DFIR-Updater/releases) page. Verify file hashes when provided.
